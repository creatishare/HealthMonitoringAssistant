# 产品需求文档 (PRD)

## 肾衰竭患者健康监测 Web 应用

**版本**：v1.1.0
**日期**：2026-03-27
**状态**：MVP开发阶段

---

## 1. 产品概述

### 1.1 产品背景

中国慢性肾病（CKD）患者约1.3亿人，其中肾衰竭患者需要长期监测多项健康指标。目前患者主要依赖纸质记录或Excel表格，存在以下痛点：
- 指标记录分散，难以追踪趋势
- 化验单数据手动录入繁琐
- 异常指标不能及时发现
- 医患沟通缺乏数据支撑
- 用药记录缺失，影响治疗跟踪

### 1.2 产品定位

面向肾衰竭患者的**个人健康数据管理工具**，帮助患者：
- 便捷记录日常监测指标
- 智能识别化验单数据
- 直观查看健康趋势
- 及时接收异常预警
- 管理用药提醒与记录

### 1.3 目标用户

| 用户类型 | 特征 | 核心需求 |
|---------|------|---------|
| **主力用户** | 50-70岁肾衰竭患者 | 简单记录、清晰展示、异常提醒、用药提醒 |
| **辅助用户** | 患者家属/护工 | 代记录、查看报告、接收预警 |
| **专业用户** | 肾内科医生 | 查看患者趋势数据（二期功能）|

### 1.4 使用场景

1. **每日居家监测**：记录体重、血压、尿量
2. **医院复查后**：拍照上传化验单，自动提取指标
3. **透析日记录**：记录透析前后体重、血压变化
4. **异常时查看**：收到预警后查看趋势图表
5. **用药管理**：设置服药提醒、记录实际服药时间
6. **血药浓度监测**：定期记录血药浓度，指导用药调整

### 2.2 功能详细说明

#### 2.2.1 用户认证模块

**注册流程**：
1. 输入手机号 → 获取验证码
2. 设置密码（6-20位，必须包含字母和数字）
3. 完善基础档案（姓名、性别、出生日期）

**登录方式**：
- 手机号 + 密码
- 手机号 + 验证码（忘记密码时使用）

#### 2.2.2 个人档案模块

**基础信息**：
- 姓名、性别、出生日期
- 身高、当前体重

**医疗档案**：
- 透析类型：无/血液透析/腹膜透析
- 干体重（透析患者）
- 基线肌酐值（用于对比）
- 诊断日期
- 原发疾病：糖尿病肾病/高血压肾病/慢性肾炎/其他

#### 2.2.3 指标录入模块

**支持录入的指标**：

| 指标名称 | 单位 | 录入方式 | 正常范围 |
|---------|------|---------|---------|
| 血清肌酐 | μmol/L | 手动/OCR | 44-133 |
| 尿素氮 | mmol/L | 手动/OCR | 2.6-7.5 |
| 血钾 | mmol/L | 手动/OCR | 3.5-5.3 |
| 血钠 | mmol/L | 手动/OCR | 136-145 |
| 血磷 | mmol/L | 手动/OCR | 0.87-1.45 |
| 血红蛋白 | g/L | 手动/OCR | 120-160 |
| 体重 | kg | 手动 | - |
| 血压（收缩/舒张） | mmHg | 手动 | <140/90 |
| 尿量 | ml/24h | 手动 | - |
| 血糖 | mmol/L | 手动/OCR | 3.9-6.1 |
| 尿酸 | μmol/L | 手动/OCR | 男性150-416/女性89-357 |
| 血药浓度 | ng/mL | 手动 | 根根据药物类型 |

**录入方式**：
1. **手动录入**：表单填写，支持历史数据补录
2. **OCR识别**：拍照化验单，自动提取数据

**血药浓度录入详情**：
- 支持药物类型选择：环孢素/他克莫司/雷帕霉素/其他
- 服药时间记录：上次服药时间
- 采血时间记录：与服药时间的间隔（如服药后2小时）
- 参考值范围：根据不同药物自动显示
  - 环孢素（C0）：100-200 ng/mL（服药前）
  - 环孢素（C2）：700-1200 ng/mL（服药后2小时）
  - 他克莫司（C0）：5-15 ng/mL（服药前）
  - 雷帕霉素（C0）：5-15 ng/mL（服药前）

#### 2.2.4 化验单OCR模块

**功能要求**：
- 支持常见医院化验单格式
- 自动识别并提取上述指标
- 置信度评估，低于80%提示用户核对
- 支持图片裁剪、旋转校正

**支持医院格式（MVP）**：
- 三甲医院标准格式
- 爱康国宾/美年大健康等体检机构

**使用流程**：
1. 点击"拍照识单"
2. 拍摄或从相册选择化验单
3. 裁剪选取化验单区域
4. 等待识别（显示进度）
5. 核对提取结果，修正错误
6. 确认保存

#### 2.2.5 用药管理模块

**功能一：用药提醒设置**

**支持设置的提醒信息**：
- 药品名称（支持选择常用药物或自定义）
- 规格（如：50mg/片）
- 每次剂量（如：2片）
- 服药频率（每日1次/每日2次/每日3次/隔日一次等）
- 服药时间（支持多时段设置）
- 提醒方式（应用内通知/短信/微信推送）

**提醒设置流程**：
1. 进入"用药管理"页面
2. 点击"添加用药"
3. 填写药品信息（名称、规格、剂量）
4. 设置服药频率和时间
5. 选择提醒方式
6. 保存设置

**提醒触发规则**：
- 按设置的时间点准时推送提醒
- 提醒内容包含：药品名称、剂量、服用时间
- 支持提前5/10/15分钟提前提醒选项
- 未记录服药状态时，每30分钟重复提醒一次（最多3次）

**功能二：用药记录**

**记录方式**：
- **确认服药**：收到提醒后点击"已服药"，自动记录服药时间
- **手动补录**：在用药记录页面补录历史服药情况
- **跳过服药**：记录本次未服药及原因（如：忘记/医生要求暂停/其他）

**记录内容**：
- 药品名称
- 计划服药时间
- 实际服药时间
- 服药状态（已服/未服/跳过）
- 备注（如：剂量调整、不良反应等）

**用药记录展示**：
- 时间线视图：按日期展示每日服药情况
- 统计视图：服药率统计、漏服次数统计
- 趋势视图：血药浓度与服药记录关联展示

#### 2.2.7 趋势图表模块

**图表类型**：
- 折线图：展示指标随时间变化趋势
- 参考线：显示正常范围上下限
- 异常标记：红色圆点标记超出范围的点

**支持指标选择**：
- 单指标视图（如只看肌酐）
- 双指标对比（如肌酐+尿素氮）
- 血药浓度+用药记录联动图

**时间范围**：
- 近7天
- 近30天
- 近3个月
- 近6个月
- 自定义范围

#### 2.2.8 预警系统

**预警级别**：

| 级别 | 颜色 | 触发条件 | 处理方式 |
|------|------|---------|---------|
| **Critical** | 红色 | 危及生命 | 弹窗+短信提醒 |
| **Warning** | 橙色 | 明显异常 | 首页卡片提醒 |
| **Info** | 蓝色 | 轻微波动 | 列表展示 |

**预警规则**：

```yaml
rules:
  - id: hyperkalemia_critical
    name: 严重高钾血症
    condition: potassium > 6.0
    level: critical
    message: "血钾严重超标（{value} mmol/L），请立即联系医生或前往急诊！"
    action: send_sms + popup

  - id: hyperkalemia_warning
    name: 高钾血症
    condition: potassium > 5.5
    level: warning
    message: "血钾偏高（{value} mmol/L），请注意饮食控制"
    action: card_alert

  - id: hyperuricemia_warning
    name: 高尿酸血症
    condition: uric_acid > 420
    level: warning
    message: "尿酸偏高（{value} μmol/L），注意控制饮食，多饮水"
    action: card_alert

  - id: drug_concentration_warning
    name: 血药浓度异常
    condition: drug_concentration超出目标范围
    level: warning
    message: "{drug_name}血药浓度{value} ng{status}建议咨询医生调整剂量"
    action: card_alert

  - id: creatinine_rise
    name: 肌酐突增
    condition: creatinine > baseline * 1.2
    level: warning
    message: "肌酐较基线上升{percent}%，建议及时就医复查"
    action: card_alert

  - id: weight_gain
    name: 体重增长过快
    condition: dialysis && weight_gain > dry_weight * 0.05
    level: warning
    message: "透析间期体重增长超过5%，请控制水分摄入"
    action: card_alert

  - id: low_hemoglobin
    name: 贫血
    condition: hemoglobin < 110
    level: info
    message: "血红蛋白偏低，建议复查血常规"
    action: list_only

  - id: missed_medication_warning
    name: 漏服提醒
    condition: scheduled_time_passed && not_recorded
    level: warning
    message: "您已错过{drug_name}的服药时间，请尽快补服或咨询医生"
    action: card_alert
```

**预警通知方式**：
- 应用内：首页卡片 + 预警列表
- Critical级别：短信通知（二期功能）

## 3. 非功能需求

### 3.1 性能要求

- 页面加载时间 < 2秒
- API响应时间 < 500ms
- OCR识别时间 < 5秒
- 支持同时在线用户 > 1000
- 用药提醒准时率 > 99%

### 3.2 安全要求

- 数据传输：TLS 1.3加密
- 密码存储：bcrypt加密（cost >= 12）
- 敏感数据：数据库字段级加密
- 会话管理：JWT + HttpOnly Cookie
- 访问控制：用户只能访问自己的数据
- 用药数据：符合HIPAA隐私保护要求

### 3.3 可用性要求

- 支持PWA离线使用（基础记录功能）
- 数据自动同步（联网时）
- 定期自动备份
- 移动端优先适配
- 用药提醒在后台稳定运行

### 3.4 无障碍要求

- 字体最小16px，重要信息18px+
- 按钮点击区域 >= 48x48px
- 颜色对比度 >= 4.5:1
- 支持屏幕阅读器
- 避免仅依靠颜色传递信息

## 8. 附录

### 8.1 术语表

- **CKD**：慢性肾病（Chronic Kidney Disease）
- **eGFR**：估算肾小球滤过率
- **Kt/V**：透析充分性指标
- **干体重**：透析后理想体重（无多余水分）
- **血药浓度（C0）**：服药前（谷浓度）
- **血药浓度（C2）**：服药后2小时（峰浓度）
- **免疫抑制剂**：环孢素、他克莫司、雷帕霉素等
