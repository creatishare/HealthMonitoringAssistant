# 页面原型 - OCR 化验单上传

**页面名称**: OCR Upload / 化验单识别
**页面路径**: `/ocr`, `/ocr/crop`, `/ocr/verify`
**版本**: v1.0.0

---

## 1. 页面概述

OCR 化验单上传模块允许用户通过拍照或从相册选择化验单图片，自动识别并提取其中的健康指标数据。减少手动录入的工作量，提高数据准确性。

### 1.1 包含页面

1. **上传页** (`/ocr`) - 拍照或选择图片
2. **裁剪页** (`/ocr/crop`) - 裁剪化验单区域
3. **核对页** (`/ocr/verify`) - 核对识别结果

### 1.2 支持识别的指标

- 血清肌酐
- 尿素氮
- 血钾、血钠、血磷
- 血红蛋白
- 尿酸
- 血糖
- 血药浓度（需手动确认药物类型）

**拍照按钮**:
- 尺寸: 200px x 200px
- 背景: `#F5F5F5`
- 边框: 2px dashed `#D9D9D9`
- 圆角: 16px
- 图标: 相机图标, 48px, `#1890FF`
- 文字: "拍摄化验单", 16px, `#595959`

**相册按钮**:
- 宽度: 200px
- 高度: 48px
- 背景: 白色
- 边框: 1px solid `#1890FF`
- 圆角: 8px
- 文字: "从相册选择", 16px, `#1890FF`

**交互**:
- 点击拍照: 调用相机
- 点击相册: 调用相册选择器
- 选择图片后跳转到裁剪页

### 2.4 使用提示

**标题**: "拍摄提示"
**提示项**:
- 每项左侧有图标（✅ 或 ❌）
- 文字: 14px, `#595959`

**提示内容**:
1. ✅ 确保光线充足，避免过暗或过亮
2. ✅ 保持化验单平整，避免褶皱
3. ✅ 确保包含完整的指标名称和数值
4. ✅ 尽量正对拍摄，避免倾斜
5. ❌ 避免手指遮挡文字
6. ❌ 避免反光和阴影

### 2.5 示例图

**可选**: 显示正确拍摄示例和错误示例对比图

- 保持比例

- 半透明遮罩覆盖框外区域

**交互**:
- 拖动边框: 调整裁剪区域

**重选按钮**:
- 图标: 返回图标

- 识别失败: 显示错误信息和重试按钮

**指标项设计**:

每项:
- 背景: 白色
- 圆角: 8px
- 内边距: 16px
- 底部边框: 1px solid `#F0F0F0`

**内容布局**:
```

│ 血清肌酐        180        μmol/L    ✓  │
│ 置信度: 95%                             │
└─────────────────────────────────────────┘
```

**字段说明**:
- 指标名称: 14px, `#595959`
- 数值: 18px, 600, `#262626`
- 单位: 12px, `#8C8C8C`

| 状态 | 图标 | 颜色 | 说明 |

| 高置信度 | ✓ | 绿色 `#52C41A` | >= 80% |
| 中置信度 | ~ | 橙色 `#FAAD14` | 60-79% |
| 低置信度 | ⚠️ | 红色 `#F5222D` | < 60% |
| 识别失败 | ✗ | 红色 `#F5222D` | 需手动输入 |

**置信度显示**:
- 文字: "置信度: XX%"
- 颜色与状态一致
- 低置信度时显示"建议核对"

## 7. 交互说明

### 7.1 流程图

```
上传页 → 裁剪页 → 识别中 → 核对页 → 保存成功
  ↓         ↓         ↓         ↓
重选      旋转      失败提示   编辑/添加
  ↓         ↓         ↓         ↓
相册      重选      重试/手动  返回列表
```

### 7.2 保存逻辑

1. 校验所有指标数值
2. 低置信度指标提示确认
3. 显示加载状态
4. 保存成功:
   - Toast 提示 "保存成功"
   - 跳转到指标列表页
5. 保存失败:
   - 显示错误信息
   - 保留核对页面

### 7.3 取消流程

**上传中取消**:
- 点击"取消上传"
- 确认弹窗
- 中断上传并返回

**核对页返回**:
- 点击返回按钮
- 确认弹窗: "确定放弃当前识别结果？"
- 确认后返回
