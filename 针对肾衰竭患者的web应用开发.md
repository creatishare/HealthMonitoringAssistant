# 针对肾衰竭患者的web应用开发

## 背景

我想要开发一款针对肾衰竭患者的健康监测web应用，现在我想要搭建多个Agent帮我完成这款产品的全部开发。下面是我设想的几个subAgent：

1. 主架构师

​	**职责**：负责全局任务分解、各子 Agent 的调用调度、以及最终的代码审查（Code Review）。

​	**输入**：产品原型描述（如：包含水分打卡、化验单 OCR、指标预警等功能）。

2. 前端工程师

​		**职责**：负责 UI/UX 实现。鉴于健康类应用需要高易用性，它将专注于：

- 响应式布局（适配患者常用的平板或手机）。
- 可视化图表（展示肌酐、尿素氮、体重波动的趋势图）。
- 交互逻辑（如分步引导式的数据录入）。

3. 后端与安全工程师

​	**职责**：负责 API 设计、数据库建模以及最关键的**隐私合规**。

- 设计符合医疗数据标准的 API 结构。
- 处理敏感的健康记录加密与权限校验。
- 集成自动化的数据校验逻辑。

4. 领域专家

   **职责**：作为“数字医生”提供逻辑支撑。

   - 预设肾衰竭相关的健康阈值逻辑（如：每日饮水量限制计算）。
   - 编写化验单数据解析的正则表达式或提示词。
   - 确保应用内的健康建议符合通用医学常识（需注明非处方建议）。
